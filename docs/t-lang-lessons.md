## 1. Dependency Management

### tproject.toml is the single source of truth for nix

| File | Purpose | Who reads it | Auto-generated? |
|------|---------|-------------|-----------------|
| `tproject.toml` | Declares R + Python + tool deps for nix | `t update` | No — edit directly |
| `flake.nix` | Nix derivation with R/Python environments | `nix develop` | **Yes** — by `t update`. NEVER edit directly. |
| `DESCRIPTION` | R package deps for non-nix users | `pak::pak()`, `R CMD check` | No — edit directly |

**Every new R dependency must be added to BOTH `tproject.toml` AND `DESCRIPTION`.**

### Why two files?

| Audience | Installs via | Reads |
|----------|-------------|-------|
| Nix/T user | `nix develop` | `tproject.toml` → `flake.nix` |
| R user (no nix) | `pak::pak("owner/repo", subdir = "packages/pkg")` | `DESCRIPTION` |

Forgetting one means:
- Missing from `tproject.toml` → R CMD check fails in nix (package not found)
- Missing from `DESCRIPTION` → non-nix users can't install

### Adding a dependency: checklist

1. Add to `tproject.toml` `[r-dependencies] packages`
2. Add to `DESCRIPTION` Imports (API dep) or Suggests (optional/pipeline dep)
3. Run `t update` (regenerates `flake.nix`)
4. `exit` + `nix develop` (picks up new flake)
5. Verify: `Rscript -e 'library(newpkg)'`

### Common mistakes

| Mistake | Consequence | Fix |
|---------|------------|-----|
| Edit `flake.nix` directly | Overwritten by next `t update` | Edit `tproject.toml` instead |
| `install.packages()` in nix | Fails (read-only nix store) | Add to `tproject.toml` |
| Add to `flake.nix` but not `tproject.toml` | Lost on `t update` | Always edit `tproject.toml` |
| Add to `tproject.toml` but not `DESCRIPTION` | Non-nix users can't install | Add to both |
| Forget `t update` after editing `tproject.toml` | `flake.nix` stale, package missing | Always `t update && exit && nix develop` |

## 2. Nix Shell Architecture

### One R environment, shared by all nodes

T creates a single `r-env` from `[r-dependencies]`. All `rn()` pipeline nodes share it. No per-node overrides.

Same for Python: single `py-env` from `[py-dependencies]`.

### Claude's dev shell ≠ project shell

| Shell | R binary | Has project deps? | Use for |
|-------|----------|-------------------|---------|
| Claude dev shell | `/nix/store/...-R-4.5.2/bin/R` | Some (impure) | Reading files, git, quarto render |
| Project shell (`nix develop`) | `/nix/store/...-R-4.5.3-wrapper/bin/R` | All | `tar_make()`, `devtools::check()`, compiled packages |

**Compiled R packages (glmnet, duckdb) SEGFAULT across shell boundaries.** The `.so` files are compiled for one R binary and crash when loaded by another.

### What runs where

| Task | Shell | Why |
|------|-------|-----|
| `tar_make()` with compiled packages | `nix develop` | ABI match |
| `devtools::check()` | `nix develop` | Needs all deps + correct R |
| `quarto render` | Claude dev shell | Only reads pre-built RDS |
| `git commit/push` | Either | No R involved |
| `t update` | `nix develop` | Needs `t` binary |

### Automation from Claude's shell

```bash
# Run R commands inside the project shell
nix develop /path/to/project --command bash -c 'Rscript -e "..."'

# Dedicated script
bash scripts/check_in_nix.sh
```

## 3. R Package in T Project

### Directory structure

```
project/
├── tproject.toml          # T project deps (single source of truth)
├── flake.nix              # Auto-generated by t update
├── src/pipeline.t         # T pipeline DAG
├── R/                     # Pipeline plan files (NOT part of R package)
│   ├── plan_vignette.R
│   ├── plan_backtest.R
│   └── plan_drif.R
├── docs/                  # Vignettes + rendered HTML
│   ├── _targets.R         # Unified targets pipeline
│   └── *.qmd
├── scripts/               # Fetch scripts, QA, automation
└── packages/
    └── historicaldata/    # The R PACKAGE
        ├── DESCRIPTION    # R package deps (for pak/CRAN)
        ├── R/             # Package source code
        ├── man/           # Roxygen docs
        ├── tests/         # testthat
        └── inst/          # Bundled data
```

### R package does NOT have its own nix shell

The R package lives inside the T project's nix environment. There is no `default.R`/`default.nix` inside `packages/historicaldata/`. Package deps come from `tproject.toml`.

### R CMD check fixes

| Issue | Fix |
|-------|-----|
| Network-dependent examples | `@examplesIf interactive()` (interim); local fallback data (proper fix, #25) |
| Network-dependent tests | `skip_if_offline()` + `curl` in Suggests |
| T extension in R package | Delete `R/_extensions/`; add `_extensions` to `.Rbuildignore` |
| pkgdown files flagged | Add `_pkgdown.yml`, `docs/`, `pkgdown/` to `.Rbuildignore` |
| "unable to verify current time" | Nix sandbox — unavoidable, not a code issue |

## 4. Never Ship Degraded Implementations

If a strategy requires package X (e.g. glmnet for elastic net):

1. **STOP** — do not write code until the package is available
2. Add to `tproject.toml` + `DESCRIPTION`
3. `t update && exit && nix develop`
4. Verify `library(X)` works
5. **THEN** write the code

**Never** write `if (!require(X)) { use_simpler_method() }`. A missing feature is honest; a degraded implementation is a lie. The DRIF vignette shipped with PCA-OLS "fallback" instead of elastic net — the OOS results looked 3x better than reality because PCA-OLS was overfitting.

## 5. HuggingFace Data Hosting

### Upload: git clone + LFS push

REST API upload endpoints are broken. The only reliable method:

```bash
HF_TOKEN=$(cat ~/.cache/huggingface/token)
TMPDIR=$(mktemp -d)
git clone "https://user:$HF_TOKEN@huggingface.co/datasets/owner/repo" "$TMPDIR/repo"
cp data/dist/file.parquet "$TMPDIR/repo/"
git -C "$TMPDIR/repo" add file.parquet
git -C "$TMPDIR/repo" commit -m "Update"
git -C "$TMPDIR/repo" push
rm -rf "$TMPDIR"
```

### Read: DuckDB hf:// protocol

```r
duckplyr::read_parquet_duckdb("hf://datasets/owner/repo/file.parquet") |>
  filter(ticker == "AAPL") |> collect()
```

34% faster than `resolve/main/` URLs. Predicate pushdown works.

### duckplyr gotchas

| Pattern | Problem | Fix |
|---------|---------|-----|
| `group_by() \|> summarise()` | "stingy duckplyr frame" | Use `.by =` inside `summarise()` |
| `as.character(min(date))` inside summarise | Type coercion fails | Move to after `collect()` |
| `tbl(con, sql("SELECT..."))` | Still raw SQL | Use `duckplyr::read_parquet_duckdb()` |

### Dataset-metadata sync

OHLCV tickers and metadata can drift. `qa_metadata_sync` target detects this.
Upload both `equity_daily.parquet` AND `metadata.parquet` together.

### yfinance volume bug

Non-US markets report incorrect volume (ranaroussi/yfinance#300). `qa_volume_sanity` target flags outliers. Don't use raw volume for cross-exchange comparisons.

## 6. Vignette Standards

### Build-info footer (mandatory)

Every vignette ends with `build_info()` from `docs/vignette_utils.R`:
- Version links to GitHub release
- Git SHA links to commit
- R version links to CRAN NEWS

### Dark mode default

All vignettes use `theme: {dark: darkly, light: cosmo}` with JS defaulting to dark.

### Click-to-zoom on plots

JS handler: click img → fullscreen overlay. Click overlay → dismiss.

### QA pipeline

| Target | Checks | Runs |
|--------|--------|------|
| `qa_summary` | Pipeline completion | Every `tar_make()` |
| `qa_metadata_sync` | OHLCV vs metadata ticker consistency | Every `tar_make()` |
| `qa_volume_sanity` | yfinance volume outliers | Every `tar_make()` |
